2. cascadeOnSchemaFail correctly reaches lastResort on double schema_fail

NVIDIA schema_fail → factory throws synthetic retryable error (line 114-120) → Groq fallback call. If Groq also returns schema_fail: the adapter returns normally (status="schema_fail", no throw), so fallbackErr is never set and canLastResort is never evaluated. Groq schema_fail returns directly to the caller with status="schema_fail", fallbackUsed=true. Haiku does NOT fire on Groq schema_fail. This is consistent behavior: only retryable throws route to lastResort, not schema_fail returns. The scenario in the brief (Groq TPD exhausted) causes a 429 throw with retryable=true, which does cascade to lastResort correctly.
