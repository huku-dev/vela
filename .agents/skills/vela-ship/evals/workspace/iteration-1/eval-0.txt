---

### Step 2: Push to develop

```bash
git push origin develop
```

---

### Step 3: Monitor CI (synchronous — do not proceed until green)

```bash
sleep 5
gh run watch $(gh run list --limit 1 --json databaseId -q '.[0].databaseId')
```

If CI fails:
```bash
gh run view <run-id> --log-failed
```
Stop. Fix. Push. Re-verify. Never proceed with red CI.

---

### Step 4: Verify Staging

Vercel auto-deploys from `develop`. Check the preview deployment is "Ready":

```bash
vercel ls --limit 3
```

Open the preview URL and confirm the signal card renders correctly. This is the staging gate — production does not proceed until staging is verified.

---

### Step 5: Pre-Production Review

Run `/ultrareview` to get multi-agent analysis of the branch before promoting. Fix any HIGH findings. Get explicit sign-off on MEDIUM findings before proceeding.

---

### Step 6: Promote to Production

```bash
git checkout main
git merge develop
git push origin main
```

Monitor production CI:
```bash
gh run watch $(gh run list --limit 1 --json databaseId -q '.[0].databaseId')
```

Keep `develop` in sync:
```bash
git checkout develop && git merge main && git push origin develop
```
