Pending Proposals Banner v2

Wireframe for E (dashboard urgency UI). Existing PendingProposalsBanner.tsx is extended in place. Banner mounts globally at Layout level, hidden on /trades. Tap behaviour unchanged: navigates to /trades. All five variants shown at mobile (375w) and one desktop reference at the bottom.

Variant 1 · Single proposal · normal urgency · T > 60m
Mobile · 375w
/dashboard
Single proposal: label includes asset · direction. Countdown rounds to Xh Ym while T > 60m. --color-status-wait-bg background, --amber-dark countdown text. Pulses every 1.6s (existing VelaLogo pulse cadence).
Variant 2 · Multiple proposals · normal urgency
Mobile · 375w
/dashboard
Multi-proposal: shortened to N trades pending so countdown fits inline without truncating the label at 375w. Countdown shows the soonest-expiring proposal; no “soonest” word needed. When that proposal expires (or gets approved), count drops by one and the countdown jumps to the next-expiring. Existing “waiting for your approval” copy is replaced because keeping it forced ellipsis or a two-row layout.
Variant 3 · Single proposal · last 60m (urgent)
Mobile · 375w
/dashboard
Background deepens to --amber-light. Pulse cadence drops to 0.9s. Countdown switches to minutes-only when < 60m.
Variant 4 · Single proposal · last 10m (critical)
Mobile · 375w
/dashboard
Border weight bumps to 3px. Countdown updates every second. Tick rate switches from 60s to 1s automatically when ≤ 10m. Still no red — red is reserved for short signals.
Variant 5 · view_only user (no countdown)
Mobile · 375w
/dashboard · view_only tier
No urgency framing because view_only users can’t act. Neutral grey bg, no pulse, no countdown. Copy reads “signal pending review” (singular pluralises naturally).
Variant 6 · full_auto user (banner hidden)
Mobile · 375w
/dashboard · full_auto tier
no banner rendered — trades auto-execute
Component returns null for full_auto users. Their proposals auto-approve, so there is nothing to surface.
Desktop reference · variant 3 (urgent) at full width
Vela  ·  dashboard
Desktop banner sits inside the content column (does not span full viewport edge-to-edge), matching how cards already render on the dashboard. Same visual logic as mobile, more horizontal breathing room.
Hidden on /trades (already shows proposals inline)
Mobile · 375w
/trades
