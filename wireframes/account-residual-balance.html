Copy decisions

  • Banner title: "Your paid plan ended" — tier-agnostic (works whether they were Standard or Premium).
  • Banner body: "Withdraw below or upgrade your plan to resume trading." — action-clear, both paths named.
  • Primary CTA: "Withdraw" — same button label as paid path, keeps muscle memory.
  • Secondary CTA: "View plans" — matches the tier-upgrade language used elsewhere in the app.
  • Sub-$3 note: "$3 minimum withdrawal (covers $2 in fees)." Explains why Withdraw is disabled without user needing to open the sheet.
  • No em dashes anywhere.

Behavior

  • No Deposit button in this path. Depositing is a paid-plan action; free users can't add funds.
  • Withdraw button routes to the existing WithdrawSheet. It re-fetches live balance from refresh-balance on mount, so if the row's cached balance is stale, the sheet still shows the true amount.
  • Withdraw is disabled when available_balance <= 0 or below the $3 tier minimum.
  • "Available" and "In trades" rows always render, matching the paid-tier BalanceCard exactly. If the user has open positions, they close normally via the system and the freed capital shifts to Available on the next sync — no user action required.
  • The tier chip / "Upgrade" link at the top of the Account page is unchanged. Path B doesn't hide those.

What Path B does NOT change

  • Backend: prod tier_config.max_daily_withdrawal_usd for free is already $1000, so process-withdrawal accepts. Zero backend edits.
  • The rest of the Account page (Recent Activity, Profile info, Subscription, Trading preferences, Notifications, Support, Legal, Log out) renders identically for both paths.
  • The existing Account-free-tier.test.ts "Trading is on paid plans" assertion continues to pass because Path A is unchanged.

Deferred (not in this patch)

  • Whether free users should also be able to deposit. Current draft hides the Deposit button, but Henry has flagged the assumption as wrong — revisit as a separate scope after this patch ships.
