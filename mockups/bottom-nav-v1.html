Bottom nav. Before and after.
Problem. Sarah's read was that inactive nav items looked disabled, not clickable. The code has no disabled state. The issue is pure contrast: inactive tabs use var(--gray-400) (#94a3b8) which reads as low-emphasis at a glance.

Current

Active tab has full ink text. Inactive tabs sit at gray-400 which looks disabled. Only a 1.5px gray-200 hairline separates the bar from content.

Signals

Proposed

Inactive tabs move to gray-700 (fully legible, unmistakably tappable). Active tab gets a signal-green pill with ink content: instant "you are here" signal, uses brand accent, reads at a glance.

Design notes
Inactive contrast. Moving to gray-700 (a WCAG-AA compliant contrast ratio on white) removes the "disabled" read entirely. Weight also drops from 600 to 500 to let the active state carry more visual emphasis on its own.
Active pill. Signal-green is Vela's brand accent, not used elsewhere in the core nav. Using it here unambiguously signals "current screen" and reinforces brand recognition. The 1.5px ink border around the pill matches the neobrutalist system.
Bar height 64px. Up from 56px. Gives the pill room to breathe without crowding icon and label, and improves tap targets (56px was slightly below iOS 44pt guideline for the inner content once labels and padding were accounted for).
Top border. 2px ink (up from 1.5px gray-200) aligns the nav with the rest of Vela's framed surfaces (plan cards, buttons, toggle). The nav no longer looks grafted onto the page.
Badges preserved. Pending-trade count on Signals and Account attention dot both continue to work. Count badge border changes to ink (or to the pill's background when active) so it remains legible on either background.
Accessibility. Pill background change is reinforced by weight change (500 → 700), so users who can't distinguish the green fill still get a clear active signal. WCAG contrast ratios check out: gray-700 on white, ink on signal-green both exceed 4.5:1.
Alternative considered
Tried a top-edge 3px signal-green indicator bar (browser-tab style) instead of a pill. Cleaner visually but failed the "instant recognition" test at small sizes, and conflicted with the ink top border. Pill won.
Implementation
Change lives in src/components/Layout.tsx lines 219 to 329. No new dependencies, no token additions required. Existing design system tokens cover all colors and spacing.
