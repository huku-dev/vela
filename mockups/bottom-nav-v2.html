Bottom nav. Three quieter active-state explorations
Shared across all three. Inactive text moves to gray-700 (up from gray-400) and weight drops 600 to 500. Top border becomes 2px ink. Bar height 64px. These three changes alone remove the "disabled" read. The options below are different ways to indicate which tab is active.
Option A

Pure contrast

No indicator graphic. Active tab is full ink + bold; inactive is gray-700 + medium. Minimal, type-forward.

Signals
Option B

Top accent bar

A 3px ink bar drops down from the top border above the active tab. Convention-friendly (Material, Figma, YouTube), unmistakable at a glance.

Signals
Option C

Signal-green underline

18px signal-green accent beneath the active label. Uses brand color in the smallest possible footprint. Quiet, ownable.

My read
Option B is the safest strong choice. The top-drop accent is the cleanest "you are here" signal at nav scale. It integrates with the 2px ink top border rather than competing. Keeps the bar type-forward and doesn't add a colored surface. Matches Vela's brutalist geometry.
Option C if you want brand ownership. The signal-green accent makes the active state distinctly Vela. Risk is it reads as small or decorative on larger phones. Usually fine on 375–430px widths though.
Option A if you want the absolute minimum. Confident, type-forward. But there is no graphic indicator at all, which means users who don't subconsciously register weight/contrast deltas may still feel ambiguity. Least safe for a broad audience.
Notes if we pick B or C
B specifics. 3px ink bar, 72% tab width (14% inset each side), seated 2px below the top border so it appears to "hang" from it. Rounded bottom corners (3px radius) so it matches the design language.
C specifics. 18px wide, 2px tall signal-green line, fully rounded, 3px below the label. Size calibrated so it visually balances a label of 6-11 characters.
