Design notes
Purpose. Catches users who abandoned Stripe checkout and would otherwise fall silently into the Free tier (current bug). Gives them a single clear choice between subscribing or explicitly skipping, while reassuring that nothing was charged.
Bottom sheet over pricing page. The pricing page is already in the user's short-term memory. Rather than routing to a new interstitial, we show the sheet on top so they can see plans faintly behind and feel the connection. Dismissing the sheet drops them straight back onto the plans, no extra nav required.
Tone and language. Avoids "checkout" wording since it reads as e-commerce and is out of sync with Vela's voice. Eyebrow is "Subscription not started" which is factual about the user's state. Title "No worries. Nothing was charged." leads with reassurance before action. No "failed" or "something went wrong" framing.
Primary CTA: "Choose a plan". Sends them back to the plans directly above. Styled as the committed action, in ink. Most bailers came close to converting and need the lowest-friction path back.
Secondary CTA: "Try free for 7 days instead". Replaces a "Skip for now" that would have dropped users into the mostly-gated Free home (which was Sarah's exact pain). Instead, we offer the trial as the step-down from subscribing. The only path to Free home is now via the trial offer screen's own "Not now, maybe later" link. To reach Free home, a user has to reject both subscribing and trialing.
Reassurance line. "No card was saved. No payment went through." One line under the copy, green check. Addresses the top anxiety after a Stripe bail (did I accidentally pay? did they keep my card?) without requiring a full trust card.
No "Resume checkout". Per your call. Stripe sessions that have been cancelled cannot be resumed; a failed-looking button would just compound the abandonment.
Does NOT appear if they bail before Stripe. This sheet is only rendered when Stripe redirects to the cancel URL. If the user taps the browser back button while Stripe is still loading, the current browser-history fix (separate item) handles that. These two fixes together close the bail hole.
Open questions
1. One-shot or always-on. If the user dismisses via "Skip for now" and later returns to /welcome/plan without ?checkout=cancelled, do we still show the sheet? My lean: no, the sheet only fires from the cancel redirect. If they return without the param, they see a normal pricing page.

2. Stripe cancel URL. Requires Stripe session config update to set cancel_url to /welcome/plan?checkout=cancelled. Edge function create-checkout-session needs this plumbed through.
