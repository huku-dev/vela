Design notes
Design system alignment. Uses Vela tokens directly (cream page, Space Grotesk headings, Inter body, JetBrains Mono prices, neobrutalist shadows, 2px ink borders, 8 and 12px radii). Copy follows voice rules. No em dashes.
Flow. Plan page is paid-only by default. High-intent users subscribe to Standard or Premium outright. Hesitant users tap "Preview Vela free for 7 days" and land on Screen 2 where the trial is framed with trust scaffolding. A final fallthrough link on Screen 2 routes the still-undecided to brief-only preview mode.
Toggle. Full-width segmented control with 2px ink border and neobrutalist shadow. "Save 17%" pill lives inside the Annual tab so the incentive is visible without requiring a click. Default: Monthly.
Plan cards. Replaces the dash-separated one-liner with a checked list. Bold feature key, muted detail. Asset count is first on Standard with the concrete breadth so users understand equities and commodities are included (addresses Sarah not knowing).
"Not ready?" card. Dashed border on warm cream. Reads as a distinct offramp, not a third plan. The word "free" pulls hesitant users without competing with the paid CTAs above.
Trial hero (Screen 2). Signal-green tint reinforces trust and low risk. Headline states the full promise in plain language.
Timeline block. The biggest trust builder. Users' main worry with card-up-front trials is forgetting and getting charged. Making the Day-5 reminder and Day-8 charge date visible removes that fear.
"You're in control" block. Addresses each common hesitation. "Cancel in one click" must actually be true in the product.
Two CTAs on Screen 2. Primary is "Start 7-day trial". Secondary is a small underlined link routing to brief-only preview. Only this secondary path leads to preview mode, so the plan page itself stays commit-focused.
Open questions
1. On day 8 default plan: Premium (what they tried) or Standard (lower friction). My lean is Premium, with a "Switch to Standard" option surfaced in the Day-5 reminder email.

2. Trial duration. 7 days aligns with the weekly signal cadence so they get one full cycle. Keeping 7?

3. Eligibility. One trial per user, or one per email. Prevents abuse via repeat signups.

4. Brief-only preview (Screen 2 fallthrough). Needs its own wireframe. Dedicated screen or an in-app preview mode with persistent upgrade bar?
