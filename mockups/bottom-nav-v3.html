Why tappability first
Reframe. Sarah's real complaint was that the other tabs looked uninteractive. In v1 and v2 the fix was inactive gray-700 plus an active indicator. v3 goes further: every tab reads as equally pressable (near-ink, 2px stroke, solid weight). The active state is a small delta layered on top, not a hierarchy shift.
2px icon strokes. Raised from 1.5px. Thin-line icons read as decorative or ghosted; 2px reads as solid objects. Applied to all nav icons consistently so no tab looks "lighter" than another.
Equal weight across tabs. Inactive and active both sit at near-ink color. Weight 500 vs 700 is the only type-level delta. Plus, option B or C add an orientation graphic. Every tab looks like the same kind of object; only one is currently on.
My read, updated
Option B remains the safest strong choice. The top bar gives strong orientation without touching the equal-weight baseline. Weight handles tappability. Bar handles orientation. Clean separation.
Option A is now viable. With all tabs at near-ink, the weight-only signal feels more intentional. Minimal, type-forward. Some users may still struggle to instantly read which tab is active, but the tappability problem is fully solved.
Option C keeps its brand ownership role. Signal-green underline works if you want the active state to feel distinctly Vela. At near-ink baseline the green pops more than in v2, which is good.
